# Code of Conduct

## Our Pledge

We want our Legal Aid Plugin project to be a welcoming, respectful, and productive space for contributors, users, and maintainers.

Everyone participating in this project is expected to treat others with respect, assume good intent where reasonable, and help keep the community constructive.

## Expected Behavior

Examples of expected behavior include:

- Being respectful and professional
- Giving constructive feedback
- Staying on topic in issues, pull requests, and discussions
- Being patient with new contributors and users
- Acknowledging mistakes and correcting them
- Respecting maintainers’ time and project scope
- Disagreeing without personal attacks
- Helping create an environment where people feel comfortable contributing

## Unacceptable Behavior

Examples of unacceptable behavior include:

- Harassment, intimidation, threats, or bullying
- Personal attacks, insults, or hostile comments
- Discriminatory language or conduct
- Sexualized language, imagery, or attention
- Deliberate disruption of discussions or project work
- Trolling, baiting, or repeated bad-faith arguments
- Publishing or threatening to publish private information
- Sharing secrets, credentials, private data, or sensitive logs
- Repeatedly demanding work from maintainers
- Abusing issue trackers, discussions, pull requests, or review comments

## Scope

This Code of Conduct applies to all project spaces, including:

- GitHub issues
- GitHub pull requests
- GitHub Discussions
- Project documentation
- Community chat or support channels, if any
- Events or spaces where someone represents the project

It also applies when behavior outside project spaces directly affects the project, its contributors, its users, or its maintainers.

## Reporting Problems

If you experience or observe behavior that violates this Code of Conduct, please report it to the maintainers.

Use one of the following reporting methods:

- Email: support@lawdroid.com
- GitHub: contact a project maintainer privately, if available

Please include as much context as you reasonably can:

- What happened
- Where it happened
- When it happened
- Who was involved
- Links, screenshots, or logs, if relevant
- Whether the issue is ongoing

Do not include private information unless it is necessary for the report.

Reports will be handled as confidentially as practical. Information may be shared with maintainers only as needed to review and respond to the situation.

## Enforcement

Maintainers are responsible for enforcing this Code of Conduct.

Depending on the situation, maintainers may take actions including:

- No action, if the report does not violate this Code of Conduct
- A private warning
- A public reminder to keep discussion constructive
- Editing or deleting comments
- Closing or locking issues, pull requests, or discussions
- Temporary restrictions from participating
- Permanent removal from project spaces
- Reporting abuse to GitHub or another relevant platform

Maintainers may act immediately when needed to protect the community, users, contributors, or the project.

## Enforcement Guidelines

Maintainers should consider the following when reviewing reports:

- The severity of the behavior
- Whether the behavior appears intentional
- Whether this is a repeated pattern
- The impact on the affected person or community
- The risk to users, contributors, maintainers, or the project
- Whether a private correction is likely to resolve the issue

The goal of enforcement is to protect the project and its community, not to punish people unnecessarily.

## Good-Faith Participation

Disagreement is allowed. Critical feedback is allowed. Bug reports, technical criticism, and strong opinions are allowed when expressed constructively.

This Code of Conduct is not intended to prevent normal technical discussion, block legitimate criticism, or require everyone to agree.

It is intended to prevent behavior that makes the project hostile, unsafe, abusive, or unproductive.

## Maintainer Conduct

Maintainers are also expected to follow this Code of Conduct.

If a maintainer violates this Code of Conduct, other maintainers should review the situation and take appropriate action.

## Appeals

If you believe an enforcement action was made in error, you may contact the maintainers with a clear explanation.

Appeals should include:

- The action being appealed
- Why you believe it was incorrect
- Any relevant context or evidence

Maintainers will review appeals in good faith, but they are not required to continue discussions that are abusive, repetitive, or made in bad faith.

## Questions

Questions about this Code of Conduct may be directed to the maintainers.
