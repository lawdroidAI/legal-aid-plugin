# Contributing

Thank you for your interest in contributing to the Legal Aid Plugin project.

This guide explains how to contribute issues, improvements, bug fixes, documentation, and plugin-related changes.

## Code of Conduct

By participating in this project, you agree to follow the project’s Code of Conduct.

Please be respectful, constructive, and patient with other contributors and maintainers.

## Ways to Contribute

You can contribute by:

- Reporting bugs
- Suggesting improvements
- Improving documentation
- Fixing bugs
- Adding tests
- Improving plugin compatibility
- Reviewing pull requests
- Helping answer questions from other users

Small contributions are welcome. You do not need to make a large change for it to be useful.

## Before You Start

Before opening a pull request, please:

1. Search existing issues and pull requests to avoid duplicate work.
2. Open an issue first for large changes, breaking changes, new features, or behavior changes.
3. Keep changes focused. One pull request should address one main thing.
4. Make sure your change is appropriate for the project’s scope.

Bug fixes, documentation improvements, and small quality-of-life improvements usually do not need prior approval.

## Reporting Bugs

When reporting a bug, please include:

- Project/plugin version
- Host application version
- Operating system and version
- Browser/runtime version, if relevant
- Steps to reproduce the issue
- Expected behavior
- Actual behavior
- Screenshots, logs, or error messages, if useful
- Any configuration that may be relevant

Please avoid sharing secrets, access tokens, private keys, personal data, or sensitive logs.

## Suggesting Features

Feature requests are welcome, but not every feature will be accepted.

A good feature request should explain:

- The problem you are trying to solve
- Why the current behavior is not enough
- The proposed solution
- Alternatives you considered
- Any compatibility, security, or privacy concerns

For plugin features, please also mention whether the change requires new permissions, new external services, or changes to the plugin manifest.

## Security Issues

Do not report security vulnerabilities in public issues or pull requests.

Please follow the process in [`SECURITY.md`](./SECURITY.md).

Security-related changes should be handled carefully and may be reviewed privately before public discussion.
